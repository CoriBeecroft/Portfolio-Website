4096

(Click here to play 4096!)

Description:

A remake of the game 2048, except instead of just manually moving tiles around to get to the 2048 tile, players have the option to specify an algorithm for moving tiles in order to get the 4096 tile.

Inspiration:

During my last semester of college I spent a lot of time playing 2048. After a while I realized I was subconsciously developing a simple algorithm for playing the game. I was pretty sure I could come up with a good enough algorithm for gameplay that it could probably get far past the 2048 tile. So I kept on developing the algorithm and testing it while I played. The algorithm was inconsistent and probably needed to be tweaked, but it was difficult to figure out what worked and what didn't work since I would make a lot of mistakes in the execution of the algorithm. I realized this was probably because I am a human and therefore not very good at that kind of thing so I decided to remake the game in such a way that I could write out and test my gameplay algorithm.

Doom Pet

Description:

A virtual pet game using graphics from id Software's original Doom games. This project is unfinished, but in its current state the game has the following functions:

  • press 'f' to feed the pet
  • press 'p' to poke the pet, to which he will respond by turning and looking at you
  • press 'd' to trigger the death animation
  • do nothing, the pet will walk back and forth in his environment

Inspiration:

I was looking for a project to work on between semesters one year and my brother suggested that I make a game using a simple game framework that he wrote. We figured it would be a good way for me to get some experience working with someone else's code and it would allow me to focus on some of the more interesting and fun parts of making a game. Though, I suspect he was just trying to get someone to test his framework because I ended up finding and fixing a lot of bugs while I was using it.

To Do:

I don't really have any plans to finish this project since I was mostly writing it to practice programming. I do still really like the idea, but a virtual pet game doesn't really make much sense as a desktop application. If I ever do finish this project, I'll probably start over and make it a mobile app.
